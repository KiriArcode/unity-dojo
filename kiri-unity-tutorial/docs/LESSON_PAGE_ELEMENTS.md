# Словарь элементов страницы урока

Справочник элементов страницы урока **сверху вниз**: как называть элемент, какие компоненты за него отвечают и где править, чтобы было проще ставить задачи (например: «здесь не применяются стили к таблице»).

---

## 1. Структура страницы (оболочка)

| Элемент | Описание | Файлы |
|--------|----------|--------|
| **Страница урока** | Layout + заголовок урока + описание + тело урока | `src/app/lessons/[slug]/page.tsx` |
| **Заголовок урока (H1)** | Рендерится из `lesson.title`, не из MDX | `page.tsx` (строка с `<h1>`) |
| **Описание урока** | Рендерится из `lesson.description` | `page.tsx` |
| **Тело урока (LessonBody)** | Вся MDX-разметка урока рендерится здесь | `LessonBody.tsx` → `MDXRemote` + `defaultMDXComponents` |

---

## 2. Базовые элементы (Markdown → обёртки в MDX)

Эти элементы появляются из обычного Markdown (заголовки, параграфы, списки, таблицы и т.д.). Им соответствуют **defaultWrappers** в `MDXComponents.tsx`.

| Название для задачи | В MDX/Markdown | Компоненты в коде | Файл | Поведение |
|---------------------|----------------|-------------------|------|-----------|
| **Заголовок 2** | `## Текст` | `h2` | MDXComponents.tsx | Заголовок секции |
| **Заголовок 3** | `### Текст` | `h3` | MDXComponents.tsx | Подзаголовок |
| **Заголовок 4** | `#### Текст` | `h4` | MDXComponents.tsx | Мелкий подзаголовок |
| **Параграф** | Обычный текст / пустая строка | `p` | MDXComponents.tsx | Абзац |
| **Список нумерованный** | `1. Пункт` | `ol` + `li` | MDXComponents.tsx | Нумерованный список |
| **Список маркированный** | `- Пункт` | `ul` + `li` | MDXComponents.tsx | Маркированный список |
| **Жирный текст** | `**текст**` | `strong` | MDXComponents.tsx | Жирное начертание |
| **Курсив** | `*текст*` | `em` | MDXComponents.tsx | Курсив |
| **Ссылка** | `[текст](url)` | `a` | MDXComponents.tsx | Гиперссылка |
| **Горизонтальная линия** | `---` | `hr` | MDXComponents.tsx | Разделитель секций |
| **Таблица урока (Markdown-таблица)** | Синтаксис `\| A \| B \|` | `table`, `thead`, `tbody`, `tr`, `th`, `td` | **MDXComponents.tsx** (строки ~71–95) | Таблица с заголовком и строками. Стили задаются в обёртках `table`, `thead`, `tbody`, `tr`, `th`, `td`. Если таблица «без стилей» — править именно эти компоненты здесь. |
| **Блок кода без компонента** | ` ``` код ``` ` | `pre` + `code` (обёртки по умолчанию) | MDXComponents.tsx | Простой блок кода без подсветки и кнопки копирования |
| **Картинка** | `![alt](src)` | `img` | MDXComponents.tsx | Изображение |

## 4. Как формулировать задачу (пример с таблицей)

**Проблема:** таблица в уроке (например, «Значение | Что происходит | Когда использовать») отображается без стилей — нет рамок, фона, отступов.

**Как назвать элемент:**  
«Таблица урока» или «Markdown-таблица в теле урока».

**Где править:**  
Стили задаются в **MDXComponents.tsx** в обёртках для таблицы:

- `table` — обёртка и классы самой таблицы
- `thead` — заголовок
- `tbody` — тело
- `tr` — строка
- `th` — ячейка заголовка
- `td` — ячейка данных

**Пример задачи:**  
«Исправить стили таблицы урока: в MDXComponents.tsx у компонентов table, thead, tbody, tr, th, td стили не применяются (или выглядят не так, как задумано). Нужно добиться видимых границ, фона заголовка и читаемых отступов в ячейках.»

Так по одному элементу можно точечно описывать, что именно «здесь не применяются стили» или что нужно изменить в поведении.
